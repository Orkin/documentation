Qovery allows you to create webhooks at organization-level so that, when an important event happens on an environment within your organization, you can get notified on external applications. For instance, you can get notifications on Slack whenever:

* an environment is created, successfully deployed, stopped or when it failed to deploy.
* a deployment started in the environment.

For more information on the Qovery-Slack integration and how to create related webhooks, see [Slack][docs.using-qovery.integration.slack]. 

You can configure all your webhooks from the [Qovery API endpoint](https://api-doc.qovery.com/#tag/Organization-Webhook/operation/listOrganizationWebHooks). To do so:

<Steps headingDepth={3}>
<ol>
<li>

Install the [Qovery CLI][docs.using-qovery.interface.cli].

</li>
<li>

Generate an [API Token][docs.using-qovery.interface.cli#generate-api-token].

</li>
<li>

Enter CURL commands using the parameters below.

</li>
</ol>
</Steps>

<Alert type="info">

Webhooks are not available in the Qovery console yet.

</Alert>

# Creating an Organization Webhook

You can create an organization webhook using the following parameters:

## Path Parameters

### organizationId

| Type    | Description                                                                                        | Mandatory  | 
|---------|----------------------------------------------------------------------------------------------------|------------|
| string  | Allows you to specify the name of the Qovery organization you want the webhook to be installed on. | Yes        |

## Query Parameters

### kind

| Type    | Description                                                  | Possible Values                                                                                                                |  Mandatory  | 
|---------|--------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------|-------------|
| string  | Allows you to define the type of webhook you want to create. | `STANDARD` <br /> `SLACK` (to push notifications to Slack, in this case the [`target_url`](#target_url) must be a Slack-compatible endpoint). | Yes         |

### target_url

| Type    | Description                                                                                                                                               |  Mandatory  | 
|---------|-----------------------------------------------------------------------------------------------------------------------------------------------------------|-------------|
| string  | Allows you to set the public HTTP or HTTPS endpoint that will receive the specified events. <br /> The target URL must start with `http://` or `https://`.| Yes         |

### target_secret

| Type    | Description                                                                                                                           |  Mandatory  | 
|---------|---------------------------------------------------------------------------------------------------------------------------------------|-------------| 
| string  | Allows you to [define a secret][docs.using-qovery.configuration.secret] to be used during each webhook call to protect sensitive data.| No          |

### description

| Type    | Description                                                                                 |  Mandatory  | 
|---------|---------------------------------------------------------------------------------------------|-------------| 
| string  | Allows you to enter a description for your webhook, so that you know exactly what it is for.| No          |

### enabled

| Type    | Description                                 |  Mandatory  | 
|---------|---------------------------------------------|-------------| 
| boolean | Allows you to turn your endpoint on or off. | No          |

### events

| Type             | Description                                                                                                               |  Possible Values                                                                                          |  Mandatory  | 
|------------------|---------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------|-------------| 
| Array of strings | Allows you to specify which events will trigger a webhook call (meaning, which events you want to get notifications for). | `DEPLOYMENT_STARTED` <br/> `DEPLOYMENT_CANCELLED` <br/> `DEPLOYMENT_FAILURE` <br/> `DEPLOYMENT_SUCCESSFUL`| Yes         |

### project_id_filter

| Type             | Description                                                                      | Mandatory  | 
|------------------|----------------------------------------------------------------------------------|------------|
| Array of strings | Allows you to filter which project(s) you want webhook calls to be triggered on. | No         |

### environment_types_filter

| Type             | Description                                                                          | Possible Values                                                   | Mandatory | 
|------------------|--------------------------------------------------------------------------------------|-------------------------------------------------------------------|-----------| 
| Array of strings | Allows you to filter which environment(s) you want webhook calls to be triggered on. |`DEPLOYMENT` <br /> `PREVIEW` <br /> `PRODUCTION` <br /> `STAGING` | No        |
